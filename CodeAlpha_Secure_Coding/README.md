# Secure Coding Review

A cybersecurity project focused on reviewing application source code to identify security vulnerabilities, analyze risks, and recommend secure coding practices for safer software development.

</div>

---

## Overview

This project focuses on performing a Secure Coding Review on a Login Authentication Website developed using HTML, CSS, and JavaScript. The review process involves analyzing source code, identifying security vulnerabilities, evaluating potential risks, and suggesting remediation techniques to improve application security.

The project demonstrates how secure coding practices can help prevent common security issues and strengthen software reliability before deployment.

## Vulnerabilities Identified

### Authentication Issues

- Hardcoded credentials
- Weak authentication logic
- Lack of backend verification

### Input Validation Issues

- Limited user input validation
- Missing input sanitization

### Password Security Issues

- Plain text password storage
- Weak password policy

### Session Security Issues

- Missing session management
- No secure authentication tokens

## Recommendations

- Remove hardcoded credentials
- Implement server-side authentication
- Encrypt and hash passwords
- Strengthen password policies
- Validate and sanitize all user inputs
- Use secure session management
- Implement HTTPS communication
- Follow secure coding standards

---

## Remediation Steps

1. Store credentials securely in a database.
2. Use password hashing algorithms.
3. Implement backend authentication mechanisms.
4. Add input validation and sanitization.
5. Configure secure session handling.
6. Perform regular security testing.
7. Review source code before deployment.
